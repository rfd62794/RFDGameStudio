# ADR-009: Shared Lua Utility Primitives

**Status:** Accepted
**Date:** July 17, 2026

## Supersedes

ADR-005 (Component systems are named patterns, not shared code).

## Context

The studio runtime loads engine primitives before game Lua source, making generic utility functions available to every game under the existing engine-loading contract. `collect`, shallow table copying, and a Lua 5.2+ `atan2` compatibility wrapper are generic utilities that had been duplicated across games.

## Decision

Shared Lua files are permitted. Generic, non-game-specific utility functions belong in `engine/primitives/` or `engine/systems/` and are loaded via the studio runtime by any game that needs them. Game-specific logic (breeding rules, combat resolution, race generation) remains in each game's own `logic.lua` unless a future ADR explicitly extends sharing further.

## Rationale

Robert's direct decision, 2026-07-17. Duplicated generic utilities (`collect`, `copy_table`, `atan2`) across multiple games were pure maintenance liability with no design benefit — unlike game-specific rules, there was never a reason for these to diverge per-game.

## Consequences

- Generic utilities may be consolidated into engine primitives when their behavior is not game-specific and their shared contract is verified by regression tests.
- The existing runtime loading contract remains authoritative: primitives are loaded before game Lua, rather than games managing module paths themselves.
- This decision does not authorize sharing game-specific rules by default. Further shared-logic consolidation requires a separate decision.
