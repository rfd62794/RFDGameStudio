# ADR-002: Lua Is the Logic Layer

**Status:** Accepted
**Date:** June 2026

## Context

Game logic needs to run in TypeScript, Python, and Rust without duplication.
Three alternatives were evaluated:

| Option | Rejected Because |
|---|---|
| JSONLogic | Insufficient expressiveness for breeding and race simulation |
| Python | Not embeddable in Rust without a large runtime; TypeScript WASM target impractical |
| JavaScript | TypeScript-only; not embeddable in Rust cleanly |

Lua has mature, small, and well-supported embedding libraries for all three target
languages: `fengari` (TypeScript), `lupa` (Python), `mlua` (Rust).

## Decision

Lua is the logic layer. The same `.lua` file runs in all three runtimes.

- TypeScript: `fengari` (compiled to WASM, runs in browser)
- Python: `lupa` (CPython extension binding to LuaJIT or Lua 5.x)
- Rust: `mlua` (safe Lua bindings for Rust)

## Consequences

- Game designers write Lua. Logic functions must be pure — no I/O, no side effects.
- The runtime seeds `math.random()` before any Lua function call. Lua never seeds
  itself.
- Lua is a design constraint, not a preference. All future game logic lives here.
- Claude can read `.lua` files and reason about game balance without execution.
