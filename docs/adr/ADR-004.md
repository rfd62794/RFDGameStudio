# ADR-004: Claude Is a First-Class Participant

**Status:** Accepted
**Date:** June 2026

## Context

The three-file format was designed to be readable by humans and by Claude.
Claude can execute tools that call the Python runtime directly.

The question was whether Claude is a code-generation assistant (outside the loop)
or a participant (inside the loop, calling tools, running simulations).

## Decision

Claude is a first-class participant at three levels:

**Director:** Reads the three files for any game and reasons about balance, gaps,
and progression without running any code. Suggests Lua function changes and
`data.yaml` constant adjustments from first principles.

**Co-designer:** Given a design prompt, generates a new three-file game definition.
(The `horse_racing` game was built in Google AI Studio in under an hour.)

**Pipeline participant:** During live sessions, Claude calls the Python runtime via
tool calls, executes Lua functions, and reports results. Can run headless simulations
(1,000 races, outcome distribution) without a browser or renderer.

The studio's design standard requires: **any game defined in three files must be
explainable to Claude in a single prompt with no additional context.** If Claude
cannot reason about the game from the three files alone, the game is under-specified.

## Consequences

- `data.yaml` must include human-readable `description` fields for all schema fields.
- `logic.lua` must include function-level comments explaining intent.
- `ui.yaml` component types must use plain English identifiers.
- Phase 3 of the roadmap is dedicated to Claude Tool Integration (MCP runtime calls).
