# ADR-003: Python Is the Phase 1 Runtime

**Status:** Accepted
**Date:** June 2026

## Context

Three runtimes are planned (Python, TypeScript, Rust). Phase 1 must ship a working
runtime without a build step. The fastest path to a verified runtime contract is:

- `rpgCore` exists in Python with 1,122 tests and proven genetics/stat patterns.
- `lupa` is mature and well-tested for CPython + Lua integration.
- Headless simulation (1,000 races) runs fastest in Python with no renderer overhead.
- No build step — `python -m pytest` is the entire feedback loop.

## Decision

Python runtime ships first. TypeScript runtime is Phase 2. Rust runtime is Phase 3.

Phase 1 deliverable: `loader.py` + `executor.py` + `runtime.py` + `validator.py`,
passing a 15/0/0 test floor, with `horse_racing` game running headless.

## Consequences

- Phase 1 is headless — no renderer. Game state is Python dicts.
- The Python runtime is the **proof-of-contract** target, not the production target.
- Test patterns established here are the template for TypeScript and Rust runtime
  tests.
- `requirements.txt` pins `lupa` and `pyyaml`. No other runtime dependencies in
  Phase 1.
